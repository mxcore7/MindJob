
### 🔌 Backend (Laravel)

The backend follows the typical MVC (Model-View-Controller) architecture of Laravel:

*   **`app/`**: The core logic of the application.
    *   **`Http/Controllers/`**: Functions that handle incoming HTTP requests (e.g., retrieving jobs, searching via API).
    *   **`Models/`**: Eloquent classes representing the database tables (e.g., `Job.php` to interact with the `jobs` table).
    *   **`Services/`**: Classes containing complex business logic (like fetching from France Travail or JSearch APIs) keeping controllers clean.
    *   **`Console/Commands/`**: Custom Artisan commands (like `php artisan jobs:fetch`).
*   **`bootstrap/`**: Scripts that initialize and start the Laravel framework.
*   **`config/`**: All backend configuration files (database connections, third-party API keys in `services.php`, mail setup).
*   **`database/`**: Contains everything related to the database schema.
    *   **`migrations/`**: PHP files that describe how to create or modify the database tables (e.g., adding the `contract_type` column).
*   **`public/`**: The web server's entry point (`index.php`) and publicly accessible assets.
*   **`resources/`**: Uncompiled front-end assets and Blade HTML templates. (Since you use Vue 3, this is mostly empty).
*   **`routes/`**: Where all the application's URLs are defined. 
    *   **`api.php`**: Where the frontend fetches data (`/api/offres`, `/api/jobs`).
    *   **`console.php`**: For scheduling tasks.
*   **`storage/`**: Files generated by Laravel, such as application cache, uploaded files, and logging (`storage/logs/laravel.log`).
*   **`tests/`**: Automated tests to verify the code works properly.
*   **`vendor/`**: Third-party PHP libraries installed by Composer (do not edit these manually).

### 🎨 Frontend (Vue 3 + Vite)

the frontend is a Single Page Application (SPA) built with Vue 3, using Pinia for state management and Vue Router for navigation.

*   **`public/`**: Static files that are served exactly as they are without going through the build tool (e.g., the site's favicon).
*   **`src/`**: The heart of the Vue application where all the front-end code lives.
    *   **`api/`**: Configuration for Axios, managing how the frontend makes HTTP requests to the Laravel backend.
    *   **`assets/`**: Uncompiled CSS files (like `main.css` for Tailwind) and static images or SVGs (like logos) that get bundled by Vite.
    *   **`components/`**: Reusable UI blocks like `JobCard.vue`, `Navbar.vue`, or buttons. These are meant to be embedded inside pages.
    *   **`layouts/`**: Components used to wrap pages (e.g., `AppLayout.vue` featuring a persistent navbar and sidebar).
    *   **`router/`**: Configuration for Vue Router (`index.js`). It maps URL paths (like `/jobs`) to specific components from the `views/` folder.
    *   **`stores/`**: Pinia state management. It holds global data (like the user session or the list of fetched jobs) so that any component can access the data without re-fetching it from the server.
    *   **`views/`**: Page-level components. Each file here corresponds to a full page seen by the user (like `JobsPage.vue` or `JobDetailPage.vue`).
    *   **`main.js`**: The main Javascript entry point. It creates the Vue app instance, injects the router and stores, and mounts it to the DOM.
*   **`node_modules/`**: Third-party JavaScript libraries installed by NPM (do not edit these manually).
